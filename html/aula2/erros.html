Erros de Principiante no HTML

1 - ESQUECER DE FECHAR TAGS

Sempre feche as tags que abrir. Uma única tag aberta pode quebrar o layout do site todo.

2 - IGNORAR A SEMÂNTICA DO HTML

Evite usar os elementos <div> ou <span> para tudo, use tags de estruturação corretas como <header>, <footer>, etc.

3 - IGNORAR O alt="" NAS IMAGENS

Se você quer que o seu site tenha usuários, um dos passos é sempre ter um alt="" dentro das imagens para garantir acessibilidade e para que o SEO(Search Engine Optimization) funcione corretamente.

4 - PULAR A HIERARQUIA DE TEXTO

Se um texto está dentro de um elemento <h1>, o próximo sempre será ou um <h2> ou um <p>, não um <h4>. Respeite a hierarquia.

5 - MISTURAR HTML E CSS

Sempre separe o HTML do CSS. Usar tags color, font ou align diretamente no HTML é uma prática defasada. Sempre separe-os em uma tag <style> dedicada ou um stylesheet separado.

6 - USAR <b> e <i> PARA ESTILIZAR

Prefira usar tags <strong> e <em>, pois são melhores para acessibilidade e melhora a pontução do site em SEOs.

7 - IDs/NOMES DUPLICADOS

Lembre-se sempre de verificar se algum outro elemento possui um mesmo ID/nome que outro. IDs e Nomes duplicados podem quebrar scripts JS e a lógica PHP.

8 - FALTA DE IDENTAÇÃO

Mesmo que estururalmente desnecessário, é uma boa prática para permitir que o código seja lido de forma correta, permitindo visualizar a hierarquia pai-filho de uma forma melhor.

9 - ANINHAÇÃO INCORRETA

Quando se aninham duas tags, por exemplo: <div><span>texto</div></span>, O elemento aberto por último será o primeiro a ser fechado. A forma correta seria: <div><span>texto</span></div>

10 - OMITIR O <!DOCTYPE html>

Quando essa tag é omitida, os navegadores podem não identificar corretamente o tipo de documento a ser processado, o que pode causar erros de renderização.
